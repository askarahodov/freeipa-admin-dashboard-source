# Управление заданиями XYOps

Портал поддерживает два действия из журнала операций:

- остановка активного задания;
- безопасный повтор завершённого задания.

## Остановка

`POST /api/integrations/runs/:runId/cancel`

Портал разрешает остановку только для live-запуска со статусом `queued`,
`running` или `unknown`. В XYOps отправляется `POST /api/app/abort_job/v1`
с сохранённым Job ID. API Key XYOps должен иметь право `abort_jobs`.
После подтверждения XYOps локальная запись получает статус `cancelled`.

## Безопасный повтор

`POST /api/integrations/runs/:runId/rerun`

Повтор не продолжает старый Job. Портал запускает новый `run_event` и связывает
новую replay-запись с исходным run ID.

Перед запуском проверяется:

1. исходный запуск завершён;
2. процесс всё ещё существует и включён;
3. сохранённая и текущая `schemaVersion` совпадают;
4. replay-параметры успешно расшифрованы;
5. для опасного процесса получено повторное подтверждение.

Если схема изменилась, API отвечает `409`, и оператор должен открыть актуальную
форму процесса.

## Хранение параметров

Параметры для повтора сохраняются отдельно от журнала операций в таблице
`operation_run_replays` и шифруются AES-GCM ключом `CONFIG_ENCRYPTION_KEY`.

Парольные поля не сохраняются даже в зашифрованном виде. Если исходный запуск
содержал заполненное поле типа `password`, повтор блокируется с требованием
ввести секрет заново через актуальную форму.

Без D1 или `CONFIG_ENCRYPTION_KEY` обычный запуск продолжает работать, но
действие safe re-run для него недоступно.

## Доступ

Оба endpoint требуют разрешение `xyops.run`. Для роли `viewer` кнопки действий
не возвращаются сервером и запрос завершается с HTTP 403.
