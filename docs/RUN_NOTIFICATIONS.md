# Уведомления о заданиях XYOps

Портал создаёт постоянные уведомления при переходе задания XYOps в один из
терминальных статусов:

- `success` — задание завершено успешно;
- `failed` — задание завершилось с ошибкой;
- `cancelled` — задание остановлено оператором.

Уведомление формируется в той же серверной точке, где сохраняется итоговый
статус `operation_runs`. Поэтому API, журнал операций и центр уведомлений
используют одно состояние.

## Хранение и дедупликация

Используются две таблицы D1:

- `operation_notifications` — одно уведомление на `run_id`;
- `operation_notification_reads` — состояние прочтения для конкретной portal
  identity.

Повторные запросы статуса и повторное сохранение терминального запуска не
создают дубликаты. В таблице сохраняются последние 500 уведомлений. Удаление
старых уведомлений также удаляет связанные записи о прочтении.

Уведомления прямых операций FreeIPA не создаются: они завершаются синхронно и
уже показывают непосредственный toast-результат. Центр уведомлений предназначен
для асинхронных Events и Workflows XYOps.

## API

Получение уведомлений текущей identity:

```text
GET /api/integrations/notifications?limit=50
```

Ответ содержит:

- `notifications` — последние уведомления;
- `unread` — общее число непрочитанных уведомлений этой identity.

Отметить отдельные уведомления прочитанными:

```text
POST /api/integrations/notifications/read
Content-Type: application/json

{"ids":["run-id"]}
```

Отметить прочитанными все сохранённые уведомления:

```json
{"all":true}
```

Чтение и изменение read-state требуют базового права `directory.read`.
Состояние прочтения одного пользователя не влияет на других пользователей.

## Безопасность

Уведомление не содержит сырой ответ XYOps, stdout, HTML и текст ошибки Job.
Сервер формирует короткое сообщение только из нормализованных полей запуска:
названия процесса, объекта операции и итогового статуса.

Текст очищается от управляющих символов и ограничивается по длине. API Key,
пароли и другие секреты в уведомления не передаются.

## Интерфейс

Кнопка с колокольчиком в верхней панели показывает число непрочитанных
уведомлений. Центр уведомлений позволяет:

- открыть связанный журнал операций;
- отметить одно уведомление прочитанным;
- отметить прочитанными все уведомления;
- включить системные Browser Notifications.

Портал опрашивает API уведомлений раз в 15 секунд. Системные уведомления
показываются только после явного разрешения пользователя и только пока портал
открыт в браузере. Это не Web Push: закрытая вкладка не получает события.

## Ограничения

Уведомления начинают создаваться после развёртывания этой версии. Исторические
завершённые задания не импортируются автоматически, чтобы не создавать большой
поток старых событий при первом запуске.
