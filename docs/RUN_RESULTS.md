# Результаты заданий XYOps

Портал отображает безопасный снимок выходных данных завершённых заданий XYOps.
XYOps остаётся источником истины: портал не исполняет плагины и не изменяет
результат, а только нормализует поддерживаемые поля завершённого Job.

## Источник данных

После завершения задания портал запрашивает `POST /api/app/get_jobs/v1` с
`verbose: true`. Полный запрос выполняется только для live-запусков со статусом
`success` или `failed`, для которых в D1 ещё нет снимка результата.

Поддерживаются поля Job:

- `description` или `message` — итоговое сообщение;
- `data` — скалярные значения и безопасные ссылки;
- `table` — таблица результата;
- `files` — выходные файлы задания.

Сырые поля `html` и `output` не отображаются и не сохраняются в таблицу
результатов портала.

## Хранение

Санитизированный снимок сохраняется в таблице D1
`operation_run_results` и связывается с записью `operation_runs` по `run_id`.
В браузер возвращаются только итоговое сообщение, виджеты значений, таблица,
проверенные ссылки и публичные метаданные файлов.

Внутренний путь файла XYOps не возвращается клиенту. Вместо него API формирует
same-origin адрес портала:

```text
GET /api/integrations/runs/:runId/files/:fileId
```

## Санитизация

Перед сохранением портал:

- удаляет ANSI-последовательности и управляющие символы;
- игнорирует ключи, похожие на пароли, секреты, токены, API keys, cookies,
  credentials, private keys, session и signatures;
- разрешает ссылки только с протоколами `http` и `https`;
- блокирует URL со встроенными логином/паролем и чувствительными query-параметрами;
- ограничивает глубину объектов, число значений, ссылок, файлов, строк и колонок;
- исключает входные файлы;
- отклоняет абсолютные пути, схемоподобный `:`, `..`, обратные слеши,
  query/hash и управляющие символы в путях файлов.

Если часть результата не помещается в безопасные лимиты, интерфейс показывает
предупреждение о сокращённом представлении.

## Скачивание файлов

Файл загружается сервером портала с использованием сохранённого XYOps API Key.
Ключ никогда не передаётся браузеру.

Прокси:

- повторно проверяет run и file ID;
- использует только заранее сохранённый относительный путь;
- после URL-нормализации требует точного совпадения origin с `XYOPS_URL`;
- не следует HTTP redirect;
- отдаёт файл как `application/octet-stream`;
- принудительно задаёт `Content-Disposition: attachment`;
- добавляет `Cache-Control: private, no-store` и
  `X-Content-Type-Options: nosniff`.

Переменная `XYOPS_RESULT_FILE_MAX_BYTES` ограничивает допустимый объявленный
размер файла. Значение по умолчанию — `52428800` байт (50 МиБ), максимальное
настраиваемое значение — 512 МиБ. Если XYOps не возвращает `Content-Length`,
файл передаётся потоково, поэтому внешний reverse proxy также должен иметь
лимит ответа и таймаут скачивания.

## Доступ

Просмотр журнала и скачивание результата требуют базового права
`directory.read`. Управляющие действия `cancel` и `rerun` по-прежнему требуют
`xyops.run`.

Для задания без поддерживаемых санитизированных данных блок результата в
интерфейсе не отображается.
