# Условная видимость полей XYOps

Портал нормализует простые и вложенные выражения `visibleWhen`, `visible_when`,
`show_when`, `condition` и `depends_on`. Одна и та же функция вычисления используется
в браузере и на сервере, поэтому скрытое поле не попадёт в `run_event`, а видимое
обязательное поле будет проверено перед запуском.

Поддерживаются листовые операторы `equals`, `notEquals`, `in`, `truthy`, `falsy` и
комбинаторы `all`/`and`, `any`/`or`, `not`. Также распознаются контракты вида
`{ operator: "and", conditions: [...] }` и `{ logic: "or", rules: [...] }`.

```json
{
  "operator": "and",
  "conditions": [
    { "field": "mode", "equals": "full" },
    {
      "or": [
        { "field": "environment", "operator": "in", "values": ["prod", "stage"] },
        { "not": { "field": "emergency", "operator": "falsy" } }
      ]
    }
  ]
}
```

Для защиты от чрезмерно сложных схем глубина выражения ограничена восемью уровнями,
а каждый логический узел — пятьюдесятью дочерними условиями. Некорректные или
неподдерживаемые выражения не сохраняются в публичной схеме; поле в таком случае
остаётся видимым, что является безопасным поведением для обязательной валидации.
