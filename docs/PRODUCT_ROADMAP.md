# Дорожная карта продукта

## Контракт продукта

Портал — сгенерированный UI и слой визуализации поверх XYOps. XYOps остается
источником истины и единственным оркестратором автоматизации. События,
Workflows, логика выполнения и интеграции не дублируются в портале.

Портал обнаруживает метаданные XYOps, строит навигацию и формы, запускает
процессы и визуализирует их жизненный цикл. FreeIPA — первый доменный модуль,
не ограничение платформы.

## Реализовано

### P0 — исполняемое ядро

- [x] Чтение и нормализация Events/Workflows из `get_events`.
- [x] Генерация форм из типов полей, требований, опций и destinations.
- [x] Маршрутизация значений в `params`, `input.data`, `workflowData`.
- [x] Запуск через `run_event` без раскрытия ключей API.
- [x] Персистентность истории операций и синхронизация активных заданий.
- [x] Персистентность настроек администратора с AES-256-GCM.
- [x] Снимок каталога и детекция изменений схем.
- [x] Навигация по категориям из XYOps.

### P1 — контракт реального XYOps

- [x] Нормализация реальных ответов XYOps (события, workflows, toolsets).
- [x] Условные поля `visible_when` / `show_when`.
- [x] Вложенные `group`, `section`, `fieldset` как группы формы.
- [x] Динамические опции через проксируемые `/api/app/…` endpoints.
- [x] Детерминированная `schemaVersion` и отчет о дрейфе маршрутов.
- [x] Обновление маршрута из источника с review-списком изменений.
- [x] Ограниченная история изменений каталога (последние 30 записей).

### P1 — визуализация выполнения

- [x] Детали запуска: этапы `stages`/`steps`/`tasks`/`nodes` из XYOps.
- [x] Автоматическая синхронизация статусов активных заданий.
- [x] Обзорные счетчики по журналу операций.

### P1 — губернаторство

- [x] RBAC: `viewer`, `operator`, `admin`.
- [x] Проверка прав на запуск и прямые мутации FreeIPA.
- [x] Токен администратора для настроек и маршрутов.
- [x] Шифрование секретов в D1.

### P2 — модули FreeIPA

- [x] Прямые мутации FreeIPA через JSON-RPC (создание, редактирование,

- [x] Fallback членства пользователей при недоступности `group_find`.

## Критерии готовности

Новое Event или Workflow XYOps попадает в каталог без изменения кода портала,
в корректной категории, со всеми поддерживаемыми полями и валидацией.
Разрешенный пользователь запускает процесс, отслеживает статус задания
и видит санитизированный результат. Ключи API и пароли не возвращаются
браузеру и не попадают в журнал.
